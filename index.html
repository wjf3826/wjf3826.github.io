关于git的简单理解

git是一个版本控制器，而github是一个数据仓库。说白了，git是一个软件，下载到本地的，用来操作github的。而github是一个网站，用来存放你的一些东西。

先说明一下本地仓库和远程仓库的区别。

从图中你可以看到，你的整个仓库其实主要是工作区和版本库，你自己平时写的文件放在工作区，你写的差不多了，然后提交到暂存区，然后觉得暂存区的也每日问题了，然后提交到相应的分支上面。主要的命令如下：

  1. 把readme.txt文件从工作区→放入暂存区

  2. 把暂存区的文件放入分支

    git commit -m “提交了一个文件readme.txt”

  3. 查看状态

  4. 把本地的分支推送到远程仓库
